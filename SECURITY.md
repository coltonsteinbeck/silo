# Security Policy

## Supported Versions

Security fixes are applied to the `main` branch. Please update to the latest release/commit for patches.

## Reporting a Vulnerability

Please report security vulnerabilities responsibly using one of the following channels (in order of preference):

1. **GitHub Security Advisory** - Use the [Security Advisories](https://github.com/coltonsteinbeck/silo/security/advisories) feature on this repository for confidential disclosure.
2. **Email** - security@coltonsteinbeck.dev (PGP key available upon request)
3. **X (Twitter)** - [@coltonsteinbeck](https://x.com/coltonsteinbeck) for non-sensitive inquiries

**What to include:**
- Clear description of the vulnerability
- Reproduction steps (minimal PoC preferred)
- Affected components and versions
- Potential impact and severity assessment

**Response timeline:**
- Initial acknowledgement: within 48 hours
- Fix coordination and testing: 5-10 business days (depends on severity)
- Public disclosure: coordinated with you after fix is released

## Scope

- Code under this repository, including infrastructure templates and workflows.
- Do not test against production deployments without prior written permission.

## Preferences

- Provide minimal PoCs; avoid automated scanners against live instances.
- If you need a safe target, use a self-hosted/local deployment.
